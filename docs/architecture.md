# Architecture

## Overview

The multi-model query tool fans out a single prompt to 7+ AI models in parallel, collects responses, and presents a unified comparison. It follows the Anthropic Harness Design pattern and the monorepo's `rlm_lib` engine conventions.

## Invocation Methods

| Method | Models | How |
|--------|--------|-----|
| CLI subprocess | claude, codex, goose, rovo-dev, gemini | `asyncio.create_subprocess_exec` |
| boto3 API | bedrock | `bedrock-runtime.invoke_model()` |
| HTTP API | ollama | `httpx.post("localhost:11434/api/generate")` |
| API fallback | claudeвҶ’anthropic, codexвҶ’openai, geminiвҶ’google | `httpx` direct to provider API |

## Key Patterns

- **Harness Design:** Follows Anthropic's three-agent pattern вҖ” Planner (spec), Generator (code), Evaluator (tests)
- **Registry-driven:** Adding a model = adding one dict entry to MODEL_REGISTRY
- **Fault isolation:** One model's failure doesn't affect others (isolated subprocesses)
- **Fallback chain:** Subscription CLI вҶ’ API Key вҶ’ Skip
- **Zero-cost telemetry:** Subscription/local models log cost_usd: 0
